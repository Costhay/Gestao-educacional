# ◈ Gestao Educacional ◈

Repositório para Solução de Gestão acadêmica para acompanhamento educacional de alunos

## ◈ Situação Problema (Ficticia) ◈

A coordenacao do SENAI vem enfrentando dificuldade para acompanhar o desempenho academico de alunos, de forma organizada, rapida e confiavel. As informacoes acabam chegando de maneiras diferentes e com ruidos, tornando o processo de analise trabalhoso e sujeito a erros. As verificacoes de aluno por aluno, para correcao dos dados e calculo de medias, feita de forma manual, aumenta risco de erros e retrabalho. Nesse cenario, fica difici: 

⬢ Visualizar o cenario academico geral  
⬢ Analisar cada caso de aluno de forma rapida e confiavel  
⬢ Identificar casos de atencao  
⬢ Reconhecer melhores desempenhos  

Nesse sentido, vamos elaborar uma aplicacao que possa solucionar essa situacao, de forma organizada e assertiva para gerar um relatorio a ser consultado, compartilhado e arquivado sobre os alunos e suas atividades registradas.

## ◈ Apresentação de Requisitos

Para que esse projeto seja executado com organização e eficiencia, vamos apresentar os requisitos a seguir.

### ◈ Regras do Negocio

RN01 - Os alunos cujas médias das atividades for menor que 7.0 serão classificados como:  Recuperação  
RN02 - Os alunos com maior media serão identificados como Top Student  

### ◈ Requisitos Funcionais

RF01 - Os dados precisam ser inseridos começando pelo nome do aluno, em seguida pelas notas. Compondo uma lista com tuplas.  
RF02 - Validar se a lista de notas não está vazia ou com dados invalidos  
RF03 - Utilizar uma estrutura de repetição que faça varredura nas notas  
RF04 - As notas devem ser processadas corretamente  
RF05 - Encontrar os casos de alunos que precisam de atenção, deve ser algo fácil e pratico  
RF06 - A solução tecnológica precisa Ser organizada  
RF07 - Deve ser gerado um relatório final para analise, compartilhamento e arquivo  

### ◈ Requisitos Não Funcionais

RNF01 - Toda a estrutura da aplicacao sera dividida em dois arquivos python: main e processamento  
RNF02 - Os arquivos ficarão em um repositório remoto para que o codigo possa ser versionado  
RNF03 - Cada funcionalidade sera atualizada e gerada por forma de uma nova branch do codigo  
RNF04 - Apos todos os testes, os arquivos serão unificados via Merge no repositorio Git, no codigo principal
